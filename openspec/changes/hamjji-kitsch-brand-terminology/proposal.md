## Why

현재 용어 체계(예: "공기업 코치", "사용자 A/B")는 기능 의미는 명확하지만 감정적 몰입과 캐릭터성이 약하다. 사용자 맥락(여자친구 애칭 "햄찌", 커플 사용성)에 맞춘 키치 톤 용어로 전환해 브랜드 기억도와 사용 지속 동기를 높일 필요가 있다. 또한 홈 화면 제품명은 하루를 함께하는 사용 맥락에 맞춰 `햄찌의 뽀짝 하루 캘린더`로 정리해야 한다.

## What Changes

- 앱 전반의 노출 문구를 "기능 중심 딱딱한 용어"에서 "햄찌 세계관 기반 키치 용어"로 재정의한다.
- 핵심 명명 규칙을 만든다: 공식 기능명(내부 식별)과 화면 노출명(브랜드 용어)을 분리해 운영한다.
- 역할 용어를 재정의한다:
  - `user A` -> 관리자 성격의 햄찌 파트너 용어
  - `user B` -> 햄찌/나햄찌 캐릭터 용어
- 도메인 이벤트 용어를 재정의한다 (예: 데이트 일정 -> `데이트데이`, 카드형 보상 -> `추억카`).
- 기념일 계산기 규칙(시작일 기준 N일/월/년 계산)과 달력 내 기념일 표기 규칙을 추가한다.
- 유행어/밈 톤을 허용하되, 과도한 저속어/비하어는 금지하는 가이드라인을 추가한다.
- 기존 기능 스펙의 표시 문구 요구사항을 갱신해 실제 UI/리포트/보상 문구에 반영한다.
- Out of Scope (MVP): 다국어 번역 체계, 개인별 톤 A/B 테스트, 자동 카피 생성 AI.

## Capabilities

### New Capabilities
- `hamjji-brand-lexicon`: 햄찌 세계관 기반 용어집, 네이밍 규칙, 금지/권장 표현, 화면 노출명 매핑 규칙을 정의한다.

### Modified Capabilities
- `user-role-selection`: 사용자 A/B 선택 UI의 라벨/설명을 햄찌 톤 용어로 변경한다.
- `study-execution`: 세션/TODO/달성률 화면의 문구와 상태 텍스트를 키치 톤으로 변경한다.
- `memory-reward-system`: 보상 카드 문구를 `추억카` 중심 브랜드 톤으로 변경하고 잠금/해금은 기본 용어로 유지한다.
- `exam-schedule-integration`: 일정 라벨(접수/시험/발표)과 캘린더 이벤트 제목 템플릿을 브랜드 톤으로 조정한다.
- `ai-interview-coaching`: 면접 모드/리포트 헤더/피드백 문구를 캐릭터 톤으로 조정한다.
- `anniversary-calendar`: 기념일 계산 결과를 일정 카드/달력 뱃지로 표기하고 반복 기념일을 자동 산출한다.

## Impact

- Affected code: UI 문자열 리소스, 라벨 매핑 테이블, 역할/보상/캘린더/면접 화면 카피, 기념일 계산 유틸/달력 렌더링.
- Affected systems: 앱 클라이언트(모바일/web) 중심, 캘린더 이벤트 제목 템플릿, 기념일 계산 로직, AI 리포트 출력 포맷.
- Dependencies: 용어집 데이터 파일, 화면 노출명 매핑 규칙, QA 문구 검수 체크리스트.
- Risk: 톤 변경으로 의미가 모호해질 수 있음 -> 기능 의미를 보존하는 보조 설명/툴팁을 함께 제공.
- External integration note: Calendar/API 연동 실패 시에도 오류 메시지는 브랜드 톤과 의미 명확성을 동시에 유지한다.
