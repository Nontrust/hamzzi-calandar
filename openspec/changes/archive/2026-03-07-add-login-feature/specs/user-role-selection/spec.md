## ADDED Requirements

### Requirement: 로그인 기반 역할 선택 진입
시스템은 로그인된 사용자 컨텍스트에서만 역할 선택 절차를 진행해야 한다. (MUST)

#### Scenario: 비로그인 상태 역할 선택 진입
- **WHEN** 비로그인 사용자가 역할 선택 화면 진입을 시도한다
- **THEN** 시스템은 로그인 화면으로 이동시키고 역할 선택을 진행하지 않아야 한다

### Requirement: 세션 기반 역할 상태 복원
시스템은 유효 세션이 있는 경우 이전에 선택한 역할 상태를 복원해야 한다. (SHALL)

#### Scenario: 앱 재진입 시 역할 복원
- **WHEN** 사용자가 유효 세션 상태에서 앱을 다시 실행한다
- **THEN** 시스템은 저장된 역할 상태를 복원하여 해당 권한 범위의 화면을 제공해야 한다

### Requirement: 로그아웃 시 역할 상태 초기화
시스템은 로그아웃 시 활성 역할 상태를 초기화해야 한다. (SHALL)

#### Scenario: 로그아웃 수행 후 재진입
- **WHEN** 사용자가 로그아웃을 완료한 뒤 앱에 다시 진입한다
- **THEN** 시스템은 이전 역할 상태를 복원하지 않고 로그인 절차부터 요구해야 한다
