## Why

현재 서비스는 사용자 식별이 역할 선택 중심으로만 동작해, 로그인 없이 주요 기능 접근이 가능하며 계정 경계가 명확하지 않다. 지금 로그인 기능을 도입해 사용자 A/B의 데이터 접근 경계를 명확히 하고, 이후 권한/개인화 기능 확장의 기반을 마련해야 한다.

## What Changes

- 이메일(또는 아이디) + 비밀번호 기반 로그인/로그아웃 흐름을 추가한다.
- 초기 데모 계정 2종(나햄찌, 햄찌 메이트)으로 로그인 가능하도록 계정 시드/초기화 절차를 정의한다.
- 로그인 성공 시 사용자 세션을 발급하고, 보호 API 및 화면 접근 시 인증 검사를 적용한다.
- 인증 실패(잘못된 비밀번호, 잠금/비활성 계정, 세션 만료) 시 표준 에러 응답과 사용자 안내 문구를 제공한다.
- **BREAKING**: 인증이 필요한 기존 API/화면은 비로그인 상태에서 접근할 수 없도록 동작이 변경된다.
- MVP 범위: 기본 로그인/로그아웃, 세션 유지, 인증 실패 처리, 데모 계정 시드.
- Out of Scope (MVP): 소셜 로그인, 비밀번호 재설정 메일, 다중 기기 세션 관리, 2FA, 계정 가입 UI.
- 외부 연동(API/Calendar) 실패 대응: 캘린더 연동 실패와 인증 실패를 분리해 처리하고, 인증은 유지한 채 연동 실패만 복구 안내/재시도로 노출한다.

## Capabilities

### New Capabilities
- `user-auth`: 아이디/비밀번호 로그인, 로그아웃, 세션 검증, 인증 에러 처리, 데모 계정 초기화 절차를 정의한다.

### Modified Capabilities
- `privacy-and-permissions`: 인증된 사용자 기준으로 접근 제어 요구사항을 강화하고, 비인증 요청 차단 요구를 추가한다.
- `server-api-foundation`: 인증 필요 API의 공통 응답/에러 코드 규약에 AUTH 계열 오류 처리 요구사항을 추가한다.
- `user-role-selection`: 로그인된 사용자 컨텍스트를 기준으로 역할 선택/유지 흐름을 수정한다.

## Impact

- Affected code:
  - `services/functions`: 로그인 핸들러, 세션 검증 미들웨어, AUTH 오류 코드 매핑
  - `apps/mobile-web`: 로그인 화면/폼, 세션 상태 관리, 보호 라우트 처리
  - `infra/migrations`: 사용자/자격증명/세션 관련 스키마(또는 기존 auth 스키마 연동)
- Affected systems:
  - 서버 인증 계층, 클라이언트 라우팅 가드, 캘린더/학습 기능 접근 정책
- Dependencies:
  - 기존 `privacy-and-permissions`, `user-role-selection`, `server-api-foundation` 스펙과 정합성
  - 데모 계정 시드 데이터(나햄찌, 햄찌 메이트) 주입 절차
- Security note:
  - 비밀번호는 코드/문서에 평문 하드코딩하지 않고, 시드/환경변수/시크릿 관리 경로로만 주입한다.
