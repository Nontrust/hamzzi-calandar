## Purpose

페이지가 분리된 라우팅 구조에서도 모바일 UI 쉘의 일관된 시각 체계와 핵심 사용 흐름을 유지하기 위한 요구사항을 확장한다.

## ADDED Requirements

### Requirement: 페이지 분리 후 UI 토큰 일관성 유지
시스템은 로그인 페이지와 홈 페이지가 분리되어도 공통 UI 토큰(색상, 간격, 타이포, 라운드)을 일관되게 적용해야 한다. (MUST)

#### Scenario: 로그인/홈 스타일 일관성
- **WHEN** 사용자가 로그인 페이지와 홈 페이지를 오간다
- **THEN** 시스템은 동일한 디자인 토큰 기준의 시각 언어를 유지해야 한다

### Requirement: 분리 페이지에서 기존 핵심 기능 가시성 보장
시스템은 페이지 분리 후에도 홈 페이지에서 역할 전환, 일정 요약, 빠른 실행, 권한 상태 등 핵심 정보와 액션을 확인할 수 있어야 한다. (MUST)

#### Scenario: 로그인 후 홈 기능 접근
- **WHEN** 사용자가 로그인 후 홈 페이지에 도착한다
- **THEN** 시스템은 기존 핵심 카드 섹션과 주요 액션을 동일한 수준으로 제공해야 한다
