# 시험 일정(Open API) 연동 설계

## 목표
관심 기관/키워드 기반으로 **채용 공고**를 수집하고, 전형 단계별로 **캘린더 이벤트**를 자동 생성한다.

## 후보 API
- data.go.kr: 공공기관 채용정보 조회(우선 검토)
- ALIO 오픈데이터: 공공기관 채용정보(OpenAPI)
- Worknet: 공채속보/채용정보 API(보조)

> 실제 연동은 **1개 API부터** 시작하고, 필드 정규화/중복 제거가 안정화되면 확장한다.

## 수집/정규화 파이프라인
1) 스케줄러가 주기적으로 호출(예: 하루 1~3회)  
2) API 응답 → `ExamEvent` 표준 스키마로 정규화  
3) 중복 제거 키: `(기관명 + 제목 + 접수기간)`  
4) 관심 키워드 필터(기관명/직렬/지역/전형 등)  
5) Google Calendar 연결 사용자는 이벤트 생성/업데이트  

## Google Calendar 이벤트 템플릿
- 제목 예:
  - `[기관] [직렬] 원서접수 시작`
  - `[기관] [직렬] 원서접수 마감`
  - `[기관] [직렬] 필기시험`
  - `[기관] [직렬] 발표`
- 설명:
  - 공고 URL
  - 준비 체크리스트(예: 서류/자소서/가산점)
  - 문의처/메모(선택)
- 알림(권장):
  - D-7 / D-3 / D-1 / D-day 오전

## 에러/예외 처리
- API 실패: 재시도(지수 백오프), 실패 로그 저장
- 필수 필드 누락: `nullable`로 저장 후 수동 편집 허용
- 캘린더 동기화 실패: `calendarSyncStatus=failed` + 재시도 버튼 제공
