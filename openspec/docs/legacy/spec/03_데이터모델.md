# 데이터 모델(초안)

> 구현 스택에 따라 DB(예: Postgres) 혹은 로컬 저장소(예: SQLite)로 매핑한다.

## 엔티티

### User
- id (uuid)
- role: `A` | `B`
- displayName
- authProvider: `apple` | `google` | `email`
- createdAt

### StudySession
- id
- userId
- startAt
- endAt
- status: `running` | `finished`
- todos: [TodoItem]
- achievementRate: 0~100
- note (선택)
- tags (선택): 예) `NCS`, `전공`, `자소서`, `면접`, `휴식`

### TodoItem
- id
- ownerUserId
- sessionId (선택) 또는 dayKey(예: `2026-03-03`)
- text
- status: `done` | `not_done`
- migratedToDate (선택): `YYYY-MM-DD`
- orderIndex

### RewardItem (추억 보상)
- id
- type: `photo` | `text` | `photo_text`
- assetUrl (사진일 때)
- text (문구)
- createdByUserId (보통 사용자 A)
- lockRuleId
- unlockedAt (nullable)

### LockRule (잠금 규칙)
- id
- conditionType: `daily_rate` | `weekly_rate` | `streak`
- threshold: 예) 70
- window: `day` | `week`
- rewardCountToUnlock: 예) 하루 1개

### ExamEvent (채용/시험 일정)
- id
- source: `alio` | `data_go_kr` | `worknet` | `manual`
- organizationName
- jobTrack (직렬/직무)
- title
- url
- applyStart (nullable)
- applyEnd (nullable)
- examDate (nullable)
- resultDate (nullable)
- interviewDate (nullable)
- calendarSyncStatus: `not_connected` | `pending` | `synced` | `failed`
- calendarEventIds (선택): 단계별 이벤트 id 저장

### InterviewSession
- id
- userId
- mode
- persona
- transcript (대화 로그)
- report (json/text)
- createdAt
