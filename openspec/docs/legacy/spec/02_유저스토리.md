# 유저 스토리 & 수용 기준

## 스토리
S1. (여자친구) 스터디를 시작할 때 오늘의 TODO 목표를 적고, 끝날 때 달성 여부를 체크하고 싶다.  
S2. (여자친구) 미달성 TODO를 내일 또는 특정 날짜로 옮겨서 이어서 하고 싶다.  
S3. (둘 다) 달성률이 기준 이상일 때만 추억(사진/문구)을 열어볼 수 있어 동기부여가 되었으면 좋겠다.  
S4. (둘 다) 공기업 채용의 원서접수/시험/발표/면접 일정을 자동으로 캘린더에 넣고 싶다.  
S5. (여자친구) AI 모의면접을 대화로 진행하고, 종료 후 피드백 리포트를 받고 싶다.  
S6. (둘 다) 유튜브 뮤직 플레이리스트를 앱에서 바로 열 수 있는 탭이 필요하다.

## 수용 기준(예시)
- 세션 종료 시: `(달성 TODO 개수 / 전체 TODO 개수) * 100`이 자동 계산되어 저장된다.
- 미달성 TODO는 한 번의 액션으로 **내일로 이월**할 수 있어야 한다.
- 특정 TODO는 달력에서 날짜를 선택하여 **원하는 날짜로 이동**할 수 있어야 한다.
- 보상(추억)은 잠금/해금 상태가 명확하게 표시된다.
- 캘린더 이벤트에는 **기관명/직렬/전형 단계(접수/시험/발표 등)**가 제목에 포함되고, 설명에 공고 링크가 포함된다.
- 모의면접은 질문 1개씩 진행되며, 종료 시 강점/개선점/개선 답변 예시/꼬리질문이 포함된 리포트를 생성한다.
