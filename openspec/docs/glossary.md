# 용어집 (Glossary)

이 문서는 OpenSpec 산출물(`proposal.md`, `design.md`, `tasks.md`, `spec.md`)에서 공통으로 사용하는 도메인 용어를 정의한다.

## 공통 원칙
- 동일 개념에는 동일 용어를 사용한다.
- 문서마다 표현이 달라지지 않도록 본 용어집의 기본 용어를 우선 사용한다.
- 규범 문장은 한국어로 작성하고 `시스템은 ... 해야 한다` 형식을 기본으로 유지한다.

## 톤 정책 (기본-키치 사이)
- 기본 톤과 키치 톤의 비율은 `기본 70 : 키치 30`을 기본값으로 유지한다.
- 기능 의미 전달이 필요한 핵심 라벨(상태/결과/오류)은 기본 용어를 우선한다.
- 키치 용어는 화면 제목/카드명/이벤트명 등 감성 접점에 제한적으로 사용한다.
- 한 화면에서 `햄` 계열 단어는 최대 1~2회까지만 사용한다.
- 오류 메시지는 `기능 의미`를 먼저 전달하고, 키치 문구는 보조로 짧게 붙인다.

## 핵심 용어
- **사용자 A**: 설정/관리/보상 콘텐츠(추억) 관리 권한을 가진 사용자 역할.
- **사용자 B**: 스터디 실행, TODO 체크, AI 모의면접을 주로 수행하는 사용자 역할.
- **활성 사용자 (Active User)**: 현재 앱 세션에서 선택되어 권한/화면 노출 기준이 되는 사용자.
- **스터디 세션 (Study Session)**: 사용자의 학습 실행 단위. `running`에서 시작해 `finished`로 종료된다.
- **TODO**: 세션 내에서 완료 여부를 체크하는 작업 항목.
- **미달성 TODO**: 세션 종료 시 `not_done` 상태인 TODO.
- **이월 (Rollover/Reschedule)**: 미달성 TODO의 목표 날짜를 내일 또는 사용자가 선택한 날짜로 이동하는 행위.
- **달성률 (Achievement Rate)**: `(완료 TODO 수 / 전체 TODO 수) * 100`으로 계산되는 비율.
- **일일 달성률**: 특정 날짜 기준으로 집계한 달성률.
- **주간 달성률**: 주 단위로 집계한 평균 달성률.
- **추억 보상 (Memory Reward)**: 사진/문구 기반 보상 콘텐츠.
- **잠금 (Locked)**: 조건 미충족 상태로 보상 열람 불가인 상태.
- **해금 (Unlocked)**: 조건 충족으로 보상 열람 가능해진 상태.
- **해금 규칙 (Unlock Rule)**: 달성률/스트릭 등 보상 해금 조건 집합.
- **일일 해금 한도 (Daily Unlock Cap)**: 하루에 해금 가능한 보상 수 제한.
- **정기 수집 (Scheduled Ingestion)**: 외부 Open API에서 일정 데이터를 주기적으로 가져오는 동작.
- **표준 스키마 매핑 (Canonical Mapping)**: 소스별 상이한 필드를 내부 공통 이벤트 모델로 변환하는 과정.
- **중복 제거 (Deduplication)**: 동일 이벤트를 하나로 병합하는 처리.
- **캘린더 동기화 (Calendar Sync)**: 표준 이벤트를 Google Calendar 이벤트로 생성/업데이트하는 동작.
- **기념일 계산기 (Anniversary Calculator)**: 기준일 기반으로 D+N/월 반복/연 반복 기념일 날짜를 계산하는 기능.
- **D-Day / D+N**: 오늘 기준 남은/지난 일수를 나타내는 상대 일수 표기.
- **업서트 (Upsert)**: 기존 이벤트가 있으면 업데이트, 없으면 생성하는 처리.
- **동기화 상태 (Sync Status)**: `pending`, `synced`, `failed` 등 동기화 진행/결과 상태.
- **실패 복구 (Failure Recovery)**: 동기화 실패 후 재시도 가능한 상태로 전환하는 처리.
- **모의면접 세션 (Interview Session)**: AI 면접 질의응답이 진행되는 대화 단위.
- **메인 질문 (Main Question)**: 한 턴에 제시되는 핵심 질문 1개.
- **꼬리질문 (Follow-up Question)**: 사용자 답변 기반으로 이어지는 추가 질문(보통 1~2개).
- **종료 리포트 (End Report)**: 면접 종료 후 생성되는 구조화 평가 결과.
- **민감 데이터 (Sensitive Data)**: 사진, 개인 메모, 면접 대화/리포트처럼 보호가 필요한 정보.
- **하드 삭제 (Hard Delete)**: 논리 삭제가 아닌 실제 데이터 제거.
- **최소 권한 요청 (Least-Privilege Permission Request)**: 기능 활성화 시점에 필요한 권한만 요청하는 정책.
- **플레이리스트 링크 (Playlist Link)**: `music.youtube.com` 또는 `youtube.com`의 재생목록 URL.

## 권장 표기
- `TODO`는 대문자 고정으로 표기한다.
- `이월`과 `Reschedule`은 같은 의미로 사용하되, 문서 기본 표기는 `이월`을 사용한다.
- `해금`은 `언락` 대신 기본 용어로 사용한다.
- `업서트`는 필요 시 괄호로 `Upsert`를 병기한다.
