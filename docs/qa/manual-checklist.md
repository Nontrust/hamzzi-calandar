# Manual QA Checklist (햄찌 용어집 기준)

## Scope
- 역할 화면(`user-role-selection`)
- 세션 화면(`study-execution`)
- 보상 화면(`memory-reward-system`)
- 일정 연동 화면(`exam-schedule-integration`)
- 기념일 화면(`anniversary-calendar`)
- 면접 리포트 화면(`ai-interview-coaching`)

## Scenarios
- [ ] 역할 선택 화면에서 `햄찌메이트/나햄찌` 라벨이 노출된다.
- [ ] 역할 라벨 하단에 권한/행동 범위 보조 설명이 함께 노출된다.
- [ ] 세션 화면에서 `집중 타임`, `오늘 할일`, `오늘 달성률` 문구가 노출된다.
- [ ] 보상 화면에서 카드명(`추억카`)과 잠금/해금 보조 문구가 함께 노출된다.
- [ ] 캘린더 이벤트 제목이 브랜드 템플릿(`데이트데이 · [기관] ...`)으로 생성된다.
- [ ] 기념일 계산기에서 기준일 기준 100일/1주년 날짜가 일관되게 계산된다.
- [ ] 31일 기준 월 반복과 윤년(2월 29일) 연 반복 케이스가 보정 규칙대로 표시된다.
- [ ] 달력에서 기념일 항목이 일반 일정과 `● 기념일` 표기로 구분된다.
- [ ] 기념일 상세에서 `기념일명 · D+N` 제목과 남은/지난 일수 문구가 함께 노출된다.
- [ ] 일정 동기화 실패 메시지에서 브랜드 톤과 재시도 안내가 동시에 표시된다.
- [ ] 면접 리포트 헤더가 `면접 피드백 노트`로 표시된다.
- [ ] 면접 피드백 문구에 금지어가 포함되면 생성이 거부/대체된다.

## Tone Balance Check (기본 70 : 키치 30)
- [ ] 화면 핵심 상태 라벨(진행/완료/실패)은 기본 용어가 우선 노출된다.
- [ ] 키치 용어는 제목/카드명/이벤트명 중심으로 제한적으로 사용된다.
- [ ] 한 화면에서 `햄` 계열 단어가 1~2회를 초과하지 않는다.
- [ ] 오류 메시지는 기능 의미(무슨 문제인지/무엇을 해야 하는지)를 먼저 전달한다.
- [ ] 키치 문구가 의미를 가리지 않고 보조 톤으로만 동작한다.

## Regression Check
- [ ] 기존 역할 전환/권한 분기 동작이 유지된다.
- [ ] 기존 일정 동기화 상태 전이(`pending/failed/retry`)가 유지된다.
- [ ] 기념일 기능 추가 후 기존 일정/보상/면접 주요 흐름에 회귀 오류가 없다.
- [ ] 기존 면접 리포트 필수 필드 저장 규칙이 유지된다.
